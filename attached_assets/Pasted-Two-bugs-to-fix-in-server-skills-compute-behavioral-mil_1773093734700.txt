--- BUG 1: Clinical call contamination ---

The transcript sample is pulling non-sales calls. Evidence: transcript
excerpts contain "my husband and I decided to take a leap of faith" and
"we're just getting moving over here" — these are patient/clinical
conversations, not sales calls.

Root cause: sampleWonTranscripts() is not enforcing deal_id linkage.

Fix: In the deals JOIN inside sampleWonTranscripts(), add this filter
to BOTH queries (the deal sample query and the per-deal call query):

  AND c.deal_id IS NOT NULL

The full condition should be:
  WHERE c.workspace_id = $1
    AND c.deal_id IS NOT NULL     ← add this
    AND c.is_internal = false
    AND c.transcript_text IS NOT NULL
    AND LENGTH(c.transcript_text) > 200

A clinical call that isn't linked to a CRM deal has no deal_id. This
single filter eliminates patient conversations from the sample entirely.

After adding the filter, log how many transcripts remain. If fewer than
5 transcripts survive, log a warning and set discoveryFailed: true so
the skill falls back to predefined milestones rather than running
discovery on an inadequate sample.

three columns are empty. The detail panel shows 0% won deals and 0× lift

     MEDIAN(days_from_open) for scored won deals where milestone = true

   assign to column 0 explicitly rather than computing from null.
