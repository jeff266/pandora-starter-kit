TASK 3 — Add "Big Deal at Risk" to the brief context

In assembleBriefData(), add a bigDealsAtRisk query that runs
alongside the existing findings query:

  SELECT
    d.id, d.name, d.amount, d.stage_normalized,
    d.rfm_grade, d.rfm_label, d.rfm_recency_days,
    d.owner_email
  FROM deals d
  WHERE d.workspace_id = $1
    AND d.stage_normalized NOT IN ('closed_won', 'closed_lost')
    AND d.rfm_grade IN ('D', 'F')
    AND d.amount >= 10000
  ORDER BY d.amount DESC
  LIMIT 5;

Add the result to the OpeningBriefData interface as:
  bigDealsAtRisk: Array<{
    id: string;
    name: string;
    amount: number;
    stage: string;
    rfmGrade: string;
    rfmLabel: string;
    daysSinceActivity: number;
    ownerEmail: string;
  }>;

"Big Deal at Risk" cards in the brief.
