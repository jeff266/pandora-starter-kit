```

function estimateQ2Coverage(
  openDeals: Deal[],
  q2Target: number,
  q1PeriodEnd: Date
): RolloverEstimate

  For each open deal:
    If close_date > q1PeriodEnd OR stage is early:
```

```
TASK 3 — Q2 coverage rollover calculation

In server/context/opening-brief.ts, add function:

  function estimateQ2Coverage(
    openDeals: OpenDeal[],
    q2Target: number,
    q1PeriodEnd: Date,
    stageWeights: Record<string, number>
  ): RolloverEstimate

Logic:
  For each open deal, classify as Q1 or Q2:
  
  Q1_CLOSE (contributes to Q1 attainment):
    close_date <= q1PeriodEnd
    AND stage_normalized IN ('contract', 'proposal')
    → 70% chance of closing in Q1 as expected
    → 30% chance of slipping to Q2
  
  Q2_PIPELINE (will be Q2 regardless):
    close_date > q1PeriodEnd
    OR stage_normalized IN ('evaluation','demo','discovery')
    → use stage weight for Q2 probability
    
  For Q2 coverage:
    Q2 weighted = 
      SUM(Q2_PIPELINE deals × stage_weight)
      + SUM(Q1_CLOSE deals × amount × 0.30)
    
  Return:
    {
      openPipelineWeighted: number,
      expectedRolloverValue: number,
      rolloverDealCount: number,
      estimatedQ2Coverage: number,  // total ÷ q2Target
      q2Target: number,
      confidence: 'estimate',
      note: string
    }

Stage weights to use (match existing pipeline weights):
  contract: 0.85
  proposal: 0.70
  demo: 0.40
  evaluation: 0.20
  discovery: 0.10

Call this function in assembleBriefData() after the
existing pipeline stats are computed.

Add estimatedQ2Coverage to OpeningBriefData interface.
```
