## The fix

In ReportViewer.tsx, find the outer div that
wraps each section card (line 1005 area).

Add a hover action strip below the section
content. The strip is hidden by default and
appears on section hover:

```typescript
// State per section: track which section
// is hovered for showing the action strip
const [hoveredSection, setHoveredSection] =
  useState<string | null>(null);

// On the section wrapper div, add:
onMouseEnter={() => setHoveredSection(section.id)}
onMouseLeave={() => setHoveredSection(null)}
```

Action strip (renders at bottom of each section):

```typescript
{hoveredSection === section.id && (
  <div style={{
    display: 'flex',
    gap: 6,
    padding: '8px 0 2px',
    borderTop: '0.5px solid #F1F5F9',
    marginTop: 12,
  }}>
    <button
      onClick={() => {
        setChartBuilderSection(section.id);
        setEditingChart(null);
      }}
      style={{
        display: 'flex',
        alignItems: 'center',

## Also fix: right-click hint text

Add a subtle hint below the section title
so users discover the right-click shortcut:

```typescript
// NOT visible on every render — only show
// if user has never right-clicked a section
// Use localStorage to track this:

const [hasUsedRightClick] = useState(
  () => localStorage.getItem(
    'pandora-rightclick-hint-dismissed'
  ) === 'true'
);

// On first render of the first section only:
{!hasUsedRightClick && sectionIndex === 0 && (
  <div style={{
    fontSize: 10,
    color: '#CBD5E1',
    marginTop: -8,
    marginBottom: 8,
  }}>
    Tip: right-click any paragraph for
    more options
  </div>
)}
```

Dismiss after first right-click:
```typescript
// In the onContextMenu handler, add:
localStorage.setItem(
  'pandora-rightclick-hint-dismissed', 'true'
);
```
