Wire Salesforce into the app so we can test the OAuth connection:

1. OAUTH ROUTES — Create server/routes/salesforce-auth.ts:

   GET /api/auth/salesforce/authorize
   - Reads SALESFORCE_CLIENT_ID and SALESFORCE_CALLBACK_URL from env
   - Generates a PKCE code_verifier (random 128 chars) and code_challenge (SHA256 + base64url)
   - Stores code_verifier in memory (simple Map keyed by state param) — we'll use DB sessions later
   - Generates a random state param for CSRF protection
   - Redirects to: https://login.salesforce.com/services/oauth2/authorize?
       response_type=code
       &client_id={SALESFORCE_CLIENT_ID}
       &redirect_uri={SALESFORCE_CALLBACK_URL}
       &scope=api refresh_token offline_access id
       &state={state}
       &code_challenge={code_challenge}
       &code_challenge_method=S256

   GET /api/auth/salesforce/callback
   - Receives ?code={auth_code}&state={state}
   - Retrieves code_verifier from memory using state
   - POSTs to https://login.salesforce.com/services/oauth2/token with:
       grant_type=authorization_code
       &code={auth_code}
       &client_id={SALESFORCE_CLIENT_ID}
       &client_secret={SALESFORCE_CLIENT_SECRET}
       &redirect_uri={SALESFORCE_CALLBACK_URL}
       &code_verifier={code_verifier}
   - Response contains: access_token, refresh_token, instance_url, id (user info URL)
   - For now, just log the response and return it as JSON so we can verify it works
   - DO NOT store tokens yet — just prove the flow works
