Replit: Agent Builder Enhancement UI

Read before starting:
1. The Agent Builder component — find where
   Goals tab and Report Structure tab render.
   Understand the current state shape for each.
   Note: these may be in a single file or split
   across tabs. Find exactly where questions
   are rendered and where issue tree sections
   are rendered.

2. The two new endpoints:
   POST /:workspaceId/agents/:agentId/
     generate-questions
   Body: { goal: string }
   Returns: { questions: [{text, rationale,
     suggested_skills}] }

   POST /:workspaceId/agents/:agentId/
     generate-sections
   Body: { goal: string, questions: string[] }
   Returns: { sections: [{title,
     standing_question, section_intent,
     action_format, primary_skill_ids,
     data_extraction_config,
     reasoning_layers, rationale}] }

3. The existing section save endpoint —
   POST /:workspaceId/agents/:agentId/issue-tree
   This is what "Use this structure" calls.
   It now accepts the new config fields.

4. How skill names are displayed in the Skills
   tab. You need to map primary_skill_ids
   (e.g. "deal-risk-review") to display names
   (e.g. "Deal Risk Assessment") in the preview.
   Find the skill registry or skills list the
   frontend already uses.

5. The Report Viewer component — find where
   sections are rendered. The question summary
   block goes above the first section.

Zero changes to Ask Pandora.
Zero changes to existing agent run behavior.

`extractVerdict(content)`:
```typescript
function extractVerdict(content?: string): string {
  if (!content) return '';
  // Get first sentence
  const firstSentence = content.split(/[.!?]/)[0]
    .trim();
  // Truncate at 100 chars
  return firstSentence.length > 100
    ? firstSentence.slice(0, 97) + '...'
    : firstSentence;
}
```

`scrollToSection(sectionId)`:
```typescript
function scrollToSection(sectionId: string) {
  const el = document.getElementById(
    `section-${sectionId}`
  );
  if (el) {
    el.scrollIntoView({
      behavior: 'smooth',
      block: 'start',
    });
  }
}
```

## Acceptance Criteria:

1. Goals tab: "✦ Suggest questions" appears
   when goal has 20+ characters

2. Clicking shows panel with 3-5 questions,
   each with rationale shown below the text

3. "Add" on a question appends to existing
   question list — does not replace

4. Already-added questions show "✓ Added"
   instead of Add button

5. "Add all questions" adds only the ones
   not already in the list

6. Panel dismisses with × button

7. Report Structure tab: teal banner with
   "✦ Generate structure" appears when goal
   exists and ≥2 questions exist

8. Clicking shows section preview with:
   - Numbered sections
   - Standing question in italics
   - Skill pills showing skill display names
   - Intent badge (teal)
   - Action format badge (gray)

9. "Use this structure" saves all sections,
   they appear in the existing section list,
   preview dismisses

10. "Edit manually" dismisses preview without
    saving anything

11. Report Viewer: question summary block
    appears above first section

12. Block is collapsed by default

13. Expanding shows questions with verdicts
    (first sentence of section content)

14. Clicking a question scrolls to that section

15. Zero regressions on existing Agent Builder
    behavior

16. Zero changes to Ask Pandora
