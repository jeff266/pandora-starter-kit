# Actions Engine: Add run_skill Internal Action Type

## Context

The internal action type infrastructure shipped in
Task #60. Four types exist: update_data_dictionary,
update_workspace_knowledge, confirm_metric_definition,
update_calibration.

Add a fifth: run_skill. Allows Ask Pandora to offer
to trigger a skill directly from an action card
instead of telling the user to navigate to the
Skills tab manually.

Observed trigger case (GrowthBook):
User asks "why are all deals graded A?" → Pandora
diagnoses deal-rfm-scoring, icp-discovery, and
deal-scoring-model have never run → currently ends
with "go to Skills tab and trigger these manually"
→ should end with three run_skill action cards.

```typescript
case 'run_skill': {
  const { skill_id, reason } = action.payload as RunSkillPayload

  // Read before snapshot
  const beforeResult = await db.query(`
    SELECT skill_id, status, last_run_at,
           completed_at, error_message
    FROM skill_runs
    WHERE workspace_id = $1
      AND skill_id = $2
    ORDER BY created_at DESC
    LIMIT 1
  `, [workspaceId, skill_id])

  const beforeSnapshot = beforeResult.rows[0] ?? null

  // Trigger the skill using the same path as
  // the manual trigger from the Skills tab.
  // Use whatever function/endpoint was found
  // in the pre-read above.
  // This should insert a new skill_runs row
  // with status = 'pending' or 'running'.

  await triggerSkill(workspaceId, skill_id)
  // OR: equivalent direct insertion if no
  // triggerSkill function exists:
  // await db.query(`
  //   INSERT INTO skill_runs
  //     (workspace_id, skill_id, status, created_at)
  //   VALUES ($1, $2, 'pending', NOW())
  // `, [workspaceId, skill_id])

  // Read after snapshot
  const afterResult = await db.query(`
    SELECT skill_id, status, created_at
    FROM skill_runs
    WHERE workspace_id = $1
      AND skill_id = $2
    ORDER BY created_at DESC
    LIMIT 1
  `, [workspaceId, skill_id])

  const afterSnapshot = afterResult.rows[0] ?? null

  // Log to knowledge_change_log
  await db.query(`
    INSERT INTO knowledge_change_log (
      workspace_id,
      target_table,
      record_key,
      before_snapshot,
      after_snapshot,
      operation,
      approved_by,
      action_id,
      created_at
    ) VALUES ($1, $2, $3, $4, $5, $6, $7, $8, NOW())
  `, [
    workspaceId,
    'skill_runs',
    skill_id,
    beforeSnapshot ? JSON.stringify(beforeSnapshot) : null,
    afterSnapshot ? JSON.stringify(afterSnapshot) : null,
    'write',
    action.approved_by ?? null,
    action.id ?? null,
  ])

  return {
    success: true,
    message: `Triggered ${skill_id} — results available in ~60 seconds`,
  }
}
```

Note on revert: run_skill is NOT revertable.
A triggered skill run can't be un-triggered.
The knowledge_change_log entry is for audit
purposes only. Do not add revert handling
for this action type.

## Step 3 — Detection in synthesis prompt

File: server/chat/orchestrator.ts
In buildAdvisorySystemPrompt() under
INTERNAL ACTION DETECTION, add:

```
OFFER run_skill when ALL of:
- Your analysis reveals a skill has never run
  (no rows in skill_runs for that skill_id)
  OR last ran more than 7 days ago
- The missing skill run directly explains the
  data quality issue being discussed
- You know the specific skill_id that should run

IMPORTANT — one card per skill:
If multiple skills need to run, surface one
separate run_skill action card for each.
Do NOT batch them into a single card.

Payload format:
{
  "action_type": "run_skill",
  "payload": {
    "skill_id": "deal-rfm-scoring",
    "reason": "Deal grading fields are empty — this skill writes risk scores to your deals"
  }
}

Known skill_ids and when to recommend them:
- deal-rfm-scoring: when deal risk scores or
  grades are missing or stale
- icp-discovery: when ICP fit scores are
  missing or all deals show the same grade
- deal-scoring-model: when deal scores haven't
  been computed or appear uniform
- pipeline-waterfall: when waterfall analysis
  is unavailable or outdated
- pipeline-hygiene: when data quality issues
  are identified in deal fields
```
