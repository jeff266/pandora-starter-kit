Now the problem is concrete. The 20-turn hard stop was a safety valve because there was no graceful context management — when the context window fills, it just dies. For calibration this is particularly painful because calibration is inherently a multi-turn interview.

**The right fix has two parts:**

**Part 1 — Sliding window with compression (fixes the hard stop)**

Instead of cutting off at 20, compress older turns as the conversation grows. The rolling window already strips tool results from older messages — extend this to also compress early exchanges into a summary paragraph injected as a system note, then drop those turns from the active window.

```
Turn 1-5:   compressed → "User confirmed: active pipeline 
             filter, Q2 target $400K, 4 reps"
Turn 6-10:  compressed → "Stale deal thresholds discussed.
             User asked to refine by stage."
Turn 11-20: full fidelity (rolling window)
```

This keeps the context window bounded without losing continuity. The conversation can run indefinitely.

**Part 2 — Calibration state machine (fixes the specific calibration case)**

Calibration is worse than a regular conversation because it has explicit state — which steps have been completed, which answers have been confirmed. That state should be persisted to the database on every turn, not held in conversation memory. If the session dies at turn 20 mid-calibration, resuming should restore from the last confirmed state, not start over.

The calibration interview already has a step structure. Each confirmed answer should write to `business_dimensions` and `workspace_config` immediately — not wait for completion. Then if the session dies, Pandora can read what's already been confirmed and pick up from the next unanswered step.Logged. Here are the two Claude Code prompts, Fix 2 first:

## Step 2 — Rule-based compression (no LLM)

Do NOT use an LLM call for compression. Use
rule-based extraction of confirmed facts from
older messages. This keeps it fast and cheap.

```typescript
function compressOlderTurns(
  messages: Message[]
): string {
  const facts: string[] = []

  for (const msg of messages) {
    if (msg.role !== 'assistant') continue
    const content = msg.content as string

    // Extract confirmed dimension definitions
    const dimMatch = content.match(
      /confirmed[:\s]+([^.]+(?:active pipeline|at.risk|commit)[^.]+)/gi
    )
    if (dimMatch) facts.push(...dimMatch)

    // Extract confirmed metrics
    const metricMatch = content.match(
      /(?:win rate|coverage|quota)[^.]*\d+[%$xk]/gi
    )
    if (metricMatch) facts.push(...metricMatch)

    // Extract rep/team confirmations
    const repMatch = content.match(
      /(?:confirmed|verified)[^.]*(?:rep|team|roster)[^.]+/gi
    )
    if (repMatch) facts.push(...repMatch)
  }

  if (!facts.length) return ''

  return `Earlier in this conversation: ${
    facts.slice(0, 5).join('. ')
  }.`
}
```
