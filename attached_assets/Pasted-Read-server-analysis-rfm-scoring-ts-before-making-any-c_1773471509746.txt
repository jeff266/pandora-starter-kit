CONTEXT — Frontera Health pipeline structure:
  AB deals:  58 open, avg $33K, range $225–$300K, 89% of pipeline value
  DB deals:   6 open, avg $34K, range $75–$150K, 9.5% of pipeline value
  DP deals:   4 open, avg $1.3K, homogeneous pricing, 0.2% of value
  Pilot/Other: 6 deals, negligible value
  
  AB and DB are economically comparable (~$33K avg each).
  DP is a different product at a different price point entirely.
  The current RFM monetary quintiles are distorted because all 
  deal types were scored together, compressing Q4 to ">$7,200"
  when the real high-value deals are $30K–$300K.

TASK 1 — Add product line detection to rfm-scoring.ts

Add a function detectProductLine(dealName: string): string that
maps deal names to product lines using these patterns:

  'Fellowship' in name → 'fellowship'
  '- AB' in name (case insensitive) → 'ab'
  '- DB' in name → 'db'  
  '- RAB' in name → 'rab'
  '- DP' in name → 'dp'
  'Pilot' in name → 'pilot'
  default → 'other'

Store the result in a new computed field or use it inline
in the scoring computation. Do NOT add a new database column
for this — derive it at compute time from deal name.
