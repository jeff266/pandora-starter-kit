## Component 1 — Dimension Badge in Report Viewer

File: client/src/pages/ReportViewer.tsx

### What to build

Each WBR/QBR section header already shows a calibration
pill from Phase 2 (teal "✓ [dimension]" or amber
"⚠ Default definition"). Extend that pill to show a
tooltip with the full filter summary on hover.

### Data available on each section

```typescript
section.dimension_key      // 'enterprise_new_biz'
section.dimension_label    // 'Enterprise New Business'
section.calibrated         // true | false
section.dimension_summary  // plain English filter text:
// "Open deals in Evaluation, Demo, Proposal, Negotiation
//  closing this quarter, excluding Pricing Review"
```

### Badge behavior

When calibrated = true:
- Pill text: "✓ [dimension_label]"
- Pill color: teal (#14B8A6), small, rounded
- On hover: show tooltip with dimension_summary text
- Tooltip style: dark background (#1E293B), white text,
  11px, max-width 280px, appears above the badge
- cursor: help on the badge

When calibrated = false:
- Pill text: "⚠ Default definition"
- Pill color: amber (#F59E0B)
- On hover: tooltip shows "This number uses Pandora's
  default pipeline filter, not a definition confirmed
  for your business. Click to calibrate."
- Clicking the amber pill opens Ask Pandora rail
  pre-loaded with "Calibrate my pipeline definition"
  (use the same mechanism as the Phase 2 amber banner
  link — check how that was implemented)

### Implementation

Use the title attribute for the tooltip in v1:
```typescript
<span
  title={section.calibrated
    ? section.dimension_summary
    : "Default definition — click to calibrate"
  }
  style={{
    cursor: section.calibrated ? 'help' : 'pointer',
    // ... existing pill styles
  }}
  onClick={!section.calibrated ? handleCalibrateClick : undefined}
>
  {section.calibrated
    ? `✓ ${section.dimension_label}`
    : '⚠ Default definition'
  }
</span>
```

If the existing Phase 2 badge already uses title
attribute — just extend it. Don't rebuild what's there.

teal. On drag-over: teal background tint.

Before uploading, show a preview table of the first 5

## Component 3 — Multi-Workspace Dimension Copy

File: client/src/pages/CalibrationSettingsTab.tsx

### What to build

A new "Copy from another workspace" section at the
bottom of the Calibration settings page. Below the
Quota Import section.

### Section header

```
── Copy from Another Workspace ───────────────────────

Copy confirmed dimension definitions from another
workspace you have access to. Copied dimensions will
be unconfirmed until you review them.
```

### Step 1 — Workspace selector

```
Source workspace:
[Dropdown — select a workspace]  [Load dimensions →]
```

The dropdown lists all workspaces the current user
has access to, excluding the current workspace.

Fetch the list from GET /api/workspaces (existing
endpoint — check what it returns and how other
workspace selectors in the app are built).

### Step 2 — Dimension selector (after loading)

After selecting a source workspace and clicking
"Load dimensions", fetch its confirmed dimensions
via GET /api/workspaces/:sourceId/calibration-status.

Show a checklist:

```
Select dimensions to copy:

☑  Active Pipeline       $2.1M confirmed  (3 conditions)
☑  Enterprise Segment    $1.4M confirmed  (2 conditions)
☐  Commit Definition     $890K confirmed  (1 condition)

Options:
☐  Copy quota targets
☑  Copy coverage/win rate targets

[Cancel]  [Copy selected (2) →]
```

Each dimension shows: label, confirmed_value formatted
as currency, condition count.

Options checkboxes map to copy_quota and copy_targets
in the request body.

### Step 3 — Compatibility warnings (if any)

Before executing the copy, show a warning panel if the
source dimensions reference custom fields that may not
exist in this workspace:

```
⚠ Compatibility warnings

  "Enterprise Segment" uses field "segment__c" which
  was not found in this workspace's CRM data. This
  condition will be copied but marked as unverified.

[Back]  [Copy anyway →]
```

Skip this step if no warnings.

### Copy execution

POST to /api/workspaces/:workspaceId/dimensions/copy-from

Request body:
```typescript
{
  source_workspace_id: string,
  options: {
    dimension_keys: string[],  // selected keys
    copy_quota: boolean,
    copy_targets: boolean,
    reset_confirmed: true      // always true from UI
  }
}
```

### Result state

```
✓ Copy complete

  3 dimensions copied
  12 stage mappings copied
  0 errors

Copied dimensions are unconfirmed. Review and confirm
each one via the calibration interview.

[Start review →]
```

"Start review" sends "Continue my calibration" to
Ask Pandora — same as the "Resume calibration" button
in CalibrationProgress.

On completion: refresh the full calibration status
display at the top of the page.
