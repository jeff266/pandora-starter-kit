```sql
CREATE TABLE custom_skills (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  workspace_id UUID NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
  
  -- Identity
  skill_id TEXT NOT NULL,           -- e.g. 'custom-renewal-risk' (slug, generated from name)
  name TEXT NOT NULL,
  description TEXT NOT NULL,        -- the "question it answers"
  category TEXT NOT NULL DEFAULT 'Custom',
  version TEXT NOT NULL DEFAULT '1.0.0',
  
  -- Data layer
  query_source TEXT NOT NULL,       -- 'saved_query' | 'inline_sql'
  saved_query_id UUID REFERENCES saved_queries(id),   -- if query_source = 'saved_query'
  inline_sql TEXT,                  -- if query_source = 'inline_sql'
  
  -- Intelligence config
  classify_enabled BOOLEAN NOT NULL DEFAULT true,
  classify_bad TEXT,                -- plain English description of "bad"
  classify_good TEXT,               -- plain English description of "good"
  synthesize_enabled BOOLEAN NOT NULL DEFAULT true,
  synthesize_tone TEXT DEFAULT 'Flag risks',   -- 'Flag risks'|'Highlight opportunities'|'Weekly summary'|'Custom'
  synthesize_custom_prompt TEXT,    -- if tone = 'Custom'
  
  -- Output / schedule
  output_slack BOOLEAN NOT NULL DEFAULT true,
  output_report BOOLEAN NOT NULL DEFAULT false,
  schedule_cron TEXT,               -- null = on-demand only
  
  -- Runtime state
  status TEXT NOT NULL DEFAULT 'active',   -- 'active' | 'disabled'
  last_run_at TIMESTAMPTZ,
  run_count INTEGER NOT NULL DEFAULT 0,
  
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  
  UNIQUE(workspace_id, skill_id)
);

CREATE INDEX idx_custom_skills_workspace ON custom_skills(workspace_id, status);

## Task 2: Dynamic Compute Step Resolution

This is the key architectural change. In server/skills/runtime.ts, find executeComputeStep() — the function that resolves computeFn by calling getToolDefinition() against the hardcoded toolRegistry.

Add a fallback path: if the computeFn name is not found in the hardcoded registry, check if it matches a saved query name in the DB.

```typescript
// In executeComputeStep(), after the existing registry lookup fails:

if (!toolDef) {
  // Fallback: check saved_queries table by name
  const savedQuery = await db.query(
    `SELECT sql, name FROM saved_queries 
     WHERE workspace_id = $1 AND name = $2 AND status = 'active'
     LIMIT 1`,
    [context.workspaceId, computeFnName]
  );
  
  if (savedQuery.rows.length > 0) {
    // Execute saved query with workspace_id as $1
    const result = await db.query(savedQuery.rows[0].sql, [context.workspaceId]);
    return { rows: result.rows, count: result.rowCount };
  }
  
  throw new Error(`Tool not found: ${computeFnName}`);
}
```

Important: saved queries executed this way are always parameterized with workspace_id = $1 only. No additional params in v1. If the query uses other params, it fails gracefully with a clear error.

## Task 3: Skill Definition Generator

Create server/skills/custom-skill-builder.ts.

```typescript
export function buildCustomSkillDefinition(row: CustomSkillRow): SkillDefinition {
  // 1. Determine computeFn name:
  //    - if query_source = 'saved_query': use the saved query's name (resolves via dynamic fallback)
  //    - if query_source = 'inline_sql': use row.skill_id as the computeFn name,
  //      and register the inline SQL into a temp saved_queries entry on first run

  // 2. Build steps array based on toggles:
  //    - Always include: Step 1 compute (computeFn → outputKey: 'raw_data')
  //    - If classify_enabled: Step 2 deepseek (generate prompt from classify_bad + classify_good)
  //    - If synthesize_enabled: Step 3 claude (generate prompt from synthesize_tone or custom_prompt)

  // 3. Generate DeepSeek prompt from plain English:
  //    Template:
  //    `You are a sales operations analyst reviewing the following data.
  //     For each item, classify as 'risk' or 'healthy'.
  //     A 'risk' item: ${classify_bad}
  //     A 'healthy' item: ${classify_good}
  //     Items: {{{json raw_data.rows}}}
  //     Respond with ONLY JSON: { "classifications": [{ "id": "...", "status": "risk|healthy", "reason": "..." }] }`

  // 4. Generate Claude prompt from tone:
  //    Map tone → instruction prefix, then append the question as the synthesis goal
  //    'Flag risks' → "You are a revenue analyst. Surface risks and required actions. Be specific — use names and numbers."
  //    'Highlight opportunities' → "You are a revenue analyst. Surface opportunities the team should act on this week."
  //    'Weekly summary' → "You are a revenue analyst writing the weekly summary. Be concise and scannable."
  //    'Custom' → use synthesize_custom_prompt directly
  //    Always append: "\n\nQuestion this skill answers: ${description}\n\n{{voiceBlock}}"

  // 5. Return SkillDefinition with:
  //    - id: row.skill_id
  //    - name: row.name
  //    - description: row.description
  //    - category: row.category (lowercase)
  //    - schedule: { cron: row.schedule_cron, trigger: 'on_demand' }
  //    - outputFormat: 'markdown'
  //    - isCustom: true  ← add this flag to SkillDefinition type if not present
}
```

## Task 4: Registry Hot-Loading

```typescript
// Load custom skills from DB at startup
export async function loadCustomSkills(workspaceId?: string): Promise<void> {
  const rows = await db.query(
    `SELECT * FROM custom_skills WHERE status = 'active'
     ${workspaceId ? 'AND workspace_id = $1' : ''}`,
    workspaceId ? [workspaceId] : []
  );
  
  for (const row of rows.rows) {
    const skillDef = buildCustomSkillDefinition(row);
    registry.register(skillDef);
  }
}

// Call at server startup (add to wherever the skill registry is initialized)
await loadCustomSkills();
```

Also export a function for hot-loading a single skill after creation (so the UI doesn't require a restart):

```typescript
export async function registerCustomSkill(skillId: string, workspaceId: string): Promise<void> {
  const row = await db.query(
    `SELECT * FROM custom_skills WHERE skill_id = $1 AND workspace_id = $2`,
    [skillId, workspaceId]
  );
  if (row.rows.length > 0) {
    registry.register(buildCustomSkillDefinition(row.rows[0]));
  }
}
