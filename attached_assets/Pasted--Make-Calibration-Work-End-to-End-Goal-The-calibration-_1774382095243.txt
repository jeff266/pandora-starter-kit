# Make Calibration Work End-to-End

## Goal

The calibration interview should work as a complete,
connected flow:

1. User triggers calibration
2. If Setup Interview already ran and stages are mapped
   in workspace_config — skip stage mapping, go straight
   to active pipeline question
3. If Setup Interview hasn't run — run stage mapping
   as step 1
4. Walk through all remaining steps with previewFilter()
   showing live deal counts at each step
5. On confirmation — write confirmed definitions to
   business_dimensions AND data_dictionary_terms
6. Calibration status updates to 'complete'

## Fix 3 — Confirmed definitions write to Data Dictionary

File: server/lib/data-dictionary.ts

After confirmDimension() saves to business_dimensions,
also write a corresponding entry to data_dictionary_terms:
```typescript
async function confirmDimension(
  workspaceId: string,
  dimensionKey: string,
  definition: DimensionDefinition
): Promise<void> {
  // Existing: save to business_dimensions
  await saveToBuisnessDimensions(workspaceId, definition)

  // NEW: also write to data_dictionary_terms
  await db.query(
    `INSERT INTO data_dictionary_terms
       (workspace_id, term, definition, source,
        category, filter_json, created_at)
     VALUES ($1, $2, $3, 'USER', $4, $5, NOW())
     ON CONFLICT (workspace_id, term)
     DO UPDATE SET
       definition = EXCLUDED.definition,
       filter_json = EXCLUDED.filter_json,
       updated_at = NOW()`,
    [
      workspaceId,
      definition.label,
      definition.description ?? buildFilterSummary(definition),
      mapDimensionToCategory(dimensionKey),
      // e.g. 'active_pipeline' → 'FILTER'
      //      'win_rate' → 'METRIC'
      JSON.stringify(definition.filter_definition),
    ]
  )
}
```

First check the actual schema of data_dictionary_terms
— column names may differ. Adapt the INSERT to match.

Do the same for saveMetricDefinition() — confirmed
metrics should auto-populate as METRIC entries in
the Data Dictionary.
```typescript
// Category mapping:
function mapDimensionToCategory(key: string): string {
  if (key.includes('pipeline') || key.includes('active'))
    return 'FILTER'
  if (key.includes('rate') || key.includes('coverage')
      || key.includes('attainment'))
    return 'METRIC'
  if (key.includes('stage'))
    return 'STAGE'
  return 'FILTER'
}
```

   enters the calibration interview — NOT general
   pipeline analysis

   calibration skips stage mapping and starts at
   active pipeline question

   and total value from previewFilter()

   (at-risk, committed, win rate, coverage)

   live numbers before asking for confirmation

   to both business_dimensions AND
   data_dictionary_terms

7. After all steps complete, calibration_status
   in workspace_config updates to 'complete'

8. Data Dictionary shows confirmed definitions
   with USER source badge

9. "All Deals" entry shows plain English,
   not "1=1 IN ()"

10. Zero TypeScript errors

11. Frontera Health workspace: test the full flow
    end to end and confirm all steps work
