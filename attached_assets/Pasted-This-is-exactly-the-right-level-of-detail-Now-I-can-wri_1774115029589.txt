## Part 2 — Seed Templates on Server

On workspace creation OR on first load if templates don't exist, seed

```typescript
export async function seedWbrQbrTemplates(
  workspaceId: string
): Promise<void> {
  // Check if WBR template already exists for this workspace
  const existing = await db.query(
    `SELECT id FROM report_templates
     WHERE workspace_id = $1
       AND created_from_template IN ('wbr_standard', 'qbr_standard')`,
    [workspaceId]
  )
  if (existing.rows.length >= 2) return  // already seeded

  // Insert WBR template
  await db.query(
    `INSERT INTO report_templates
       (workspace_id, name, description, sections, cadence,
        formats, created_from_template, is_active)
     VALUES ($1, $2, $3, $4, $5, $6, $7, $8)
     ON CONFLICT DO NOTHING`,
    [
      workspaceId,
      'Weekly Business Review',
      'Operational pipeline review for RevOps and sales leadership. 15-20 minutes. Lead with the number, close with actions.',
      JSON.stringify(WBR_SECTIONS),
      'weekly',
      JSON.stringify(['pdf']),
      'wbr_standard',
      true,
    ]
  )

  // Insert QBR template
  await db.query(
    `INSERT INTO report_templates
       (workspace_id, name, description, sections, cadence,
        formats, created_from_template, is_active)
     VALUES ($1, $2, $3, $4, $5, $6, $7, $8)
     ON CONFLICT DO NOTHING`,
    [
      workspaceId,
      'Quarterly Business Review',
      'Strategic quarter review for RevOps presenting to sales leadership and executives. 45-60 minutes. What happened, why, and what changes.',
      JSON.stringify(QBR_SECTIONS),
      'quarterly',
      JSON.stringify(['pdf']),
      'qbr_standard',
      true,
    ]
  )
}

## Part 4 — Generation Endpoint Update

File: server/routes/reports.ts — POST /reports/:reportId/generate

Accept two new optional body params:

```typescript
{
  period_label?: string,    // "Week of March 17, 2026" or "Q1 2026"

}
```

When generating a WBR or QBR:
1. Set report_documents.document_type to 'wbr' or 'qbr'
2. Set report_documents.week_label to period_label
3. For each section in the template, pull the most recent skill_run
   output for each skill in section.skills and use it to populate
   the section content
4. If a skill has no recent run, set the section content to a
   degraded placeholder:
   ```

   Run it from the Skills page to populate this section.
   ```

The content generation logic already exists — this just ensures
document_type and week_label are correctly set from the new params.
