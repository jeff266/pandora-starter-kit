Bug 1: Save not persisting
The POST to /reports/:reportId/annotations is either failing
silently or the saved annotation isn't being added to
docAnnotations state. 

Check:
- Is the endpoint mounted correctly and returning 200 + saved row?
- After onAnnotationSave resolves, is the returned annotation
  being appended to docAnnotations in ReportViewer?
- Is report_document_id being sent correctly in the POST body?
  It should be reportDocument.id, not undefined.
- Add console.log at the start of onAnnotationSave and after
  the fetch to confirm what's being sent and received.
Fix whatever is broken in the save flow.

Bug 2: Blue border not appearing after save
The Override visual indicator (blue left border) only renders
when docAnnotations contains a matching annotation for that
section_id + paragraph_index. If save is broken, this won't
show. But also check:
- In AnnotatableSection, is the annotations prop actually
  being filtered correctly from docAnnotations?
- Is the paragraph_index matching? The index used when saving
  must match the index used when rendering.
- After fix to Bug 1, verify border appears without additional
  changes. If it still doesn't appear after save works,
  the annotations filter or visual render logic needs fixing.

Bug 3: Override UX — pre-fill textarea with original text

- Remove the "Replacing:" preview div entirely
- When Override tab is selected, pre-fill the textarea with
