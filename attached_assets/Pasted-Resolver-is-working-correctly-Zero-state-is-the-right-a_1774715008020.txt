**ConditionSource resolution**

Build a private function `resolveConditionSource(source: ConditionSource, wi: WorkspaceIntelligence): { value: unknown; confidence: ConfidenceLevel; resolved: boolean }`.

```typescript
// literal → value as-is, confidence CONFIRMED, resolved true
// config_ref → traverse WI object by dot-path string
//   e.g. 'pipeline.active_stages' → wi.pipeline.active_stages
//   if path resolves to null/undefined/empty array → resolved false, confidence UNKNOWN
//   if path resolves to a value → resolved true
//   confidence = infer from domain:
//     pipeline.* → wi.readiness.by_domain.pipeline > 50 ? CONFIRMED : INFERRED
//     segmentation.* → wi.readiness.by_domain.segmentation > 50 ? CONFIRMED : INFERRED
//     taxonomy.* → wi.readiness.by_domain.taxonomy > 50 ? CONFIRMED : INFERRED
//     metrics.* → look up wi.metrics[key].confidence
//     otherwise → INFERRED
// metric_ref → wi.metrics[metric_key].last_computed_value
//   if null → resolved false, confidence UNKNOWN
//   if present → resolved true, confidence = wi.metrics[metric_key].confidence
// date_scope → convert to { start: Date, end: Date } based on current date
//   current_period → start of current quarter to today
//   prior_period → full prior quarter
//   rolling_30/60/90 → today minus N days
//   ytd → Jan 1 of current year to today
//   resolved always true, confidence CONFIRMED
```

Use a dot-path resolver utility — do not hardcode every possible path:

```typescript
function resolveDotPath(obj: unknown, path: string): unknown {
  return path.split('.').reduce((acc, key) => {
    if (acc === null || acc === undefined) return undefined;
    return (acc as Record<string, unknown>)[key];
  }, obj);
}
```

**SQL building logic**

Private function `buildSQL(definition: QueryDefinition, resolvedConditions: ResolvedCondition[], wi: WorkspaceIntelligence): { sql: string; params: unknown[] }`.

```typescript
// 1. Start params array with workspaceId as $1 — always first
// 2. SELECT clause: build from aggregation.fn and aggregation.field
//    COUNT(*) → SELECT COUNT(*)
//    SUM(amount) → SELECT SUM(amount)
//    AVG(close_date - create_date) → special case for days calculation
// 3. FROM clause: ENTITY_TABLE[definition.entity] aliased as primary table
// 4. JOIN clauses: from definition.joins if present
//    ALSO: auto-inject company join if:
//      wi.pipeline.coverage_requires_segmentation = true
//      AND entity = 'deal'
//      AND no company join already defined
//      AND definition.group_by is not already set
// 5. WHERE clause:
//    Always first condition: workspace_id = $1
//    Then each resolved condition as additional AND clauses
//    Use $N parameterized placeholders, increment for each param
//    operator mapping:
//      eq → = $N
//      neq → != $N
//      in → = ANY($N) — pass array as single param
//      not_in → != ALL($N) — pass array as single param
//      gt/lt/gte/lte → >/</>=/<= $N
//      is_null → IS NULL (no param)
//      not_null → IS NOT NULL (no param)
// 6. GROUP BY clause: from definition.group_by if present
//    If auto-injected company join: add company segment field to GROUP BY
//    The segment field = wi.segmentation.dimensions[first confirmed dimension].crm_field
// 7. date_scope: add to WHERE as additional date range condition
//    field from definition.date_scope.field, values from resolved date_scope
```

**Confidence propagation**

Track confidence across all resolved sources:

```typescript
function mergeConfidence(levels: ConfidenceLevel[]): ConfidenceLevel {
  if (levels.includes('UNKNOWN')) return 'UNKNOWN';
  if (levels.includes('INFERRED')) return 'INFERRED';
  return 'CONFIRMED';
}
```

Overall query confidence = merge of all condition source confidences.

**CompiledQuery output**

```typescript
return {
  sql: sql,                    // null if any required ref unresolved
  params: params,
  confidence: mergeConfidence(allConfidenceLevels),
  unresolved_refs: unresolvedPaths,   // config_ref paths that returned undefined
  fallback_used: unresolvedPaths.length > 0,
  warnings: warnings,          // human readable strings e.g. "pipeline.active_stages is empty — no stage filter applied"
};
```

If any `config_ref` is unresolved AND the condition is required (not optional) → set `sql = null`. A skill receiving `sql: null` must not execute the query.

Mark a condition as optional by wrapping in `{ optional: true, ...condition }` — extend the Condition type if needed.

**Auto-inject logic for Frontera**

This is the specific bug fix. When:
- `definition.entity === 'deal'`
- `wi.pipeline.coverage_requires_segmentation === true`
- No join to `company` already in `definition.joins`

Then:
1. Auto-inject: `JOIN companies ON deals.company_id = companies.id`
2. Find the first confirmed dimension in `wi.segmentation.dimensions` where `entity === 'company'`
3. Auto-inject that dimension's `crm_field` into `GROUP BY`
4. Add warning: `"Segmentation auto-injected: Frontera requires pipeline broken down by segment"`
