## Fix A — Section rendering

If the renderer only renders section[0] or only the
headline: iterate all sections[] and render each one.

Each section in the WBR gets:
1. A bold section heading (Heading 2 style)
2. The section narrative as body text paragraphs
3. Metrics as a table if section.metrics[] exists
4. A horizontal rule between sections

```typescript
// For each section in report_document.sections:
for (const section of sections) {
  // Section heading
  children.push(new Paragraph({
    heading: HeadingLevel.HEADING_2,
    spacing: { before: 360, after: 160 },
    children: [new TextRun({
      text: section.title ?? section.label ?? '',
      bold: true, size: 28, font: 'Arial'
    })]
  }))

  // Narrative paragraphs
  // Split on double newline for paragraph breaks
  const paragraphs = (section.narrative ?? '')
    .split('\n\n')
    .filter(p => p.trim())

  for (const para of paragraphs) {
    children.push(new Paragraph({
      spacing: { before: 60, after: 120 },
      children: [new TextRun({
        text: para.replace(/\*\*/g, '').trim(),
        size: 22, font: 'Arial'
      })]
    }))
  }

  // Metrics table if present
  if (section.metrics?.length) {
    children.push(buildMetricsTable(section.metrics))
  }

  // Section divider
  children.push(new Paragraph({
    spacing: { before: 200, after: 200 },
    border: { bottom: {
      style: BorderStyle.SINGLE, size: 2, color: 'D1D5DB'
    }},
    children: []
  }))
```

## Fix B — Table column width bug

Per the docx skill rules — tables REQUIRE dual widths:
1. columnWidths array on the Table object
2. width on EVERY TableCell individually
Both must be set in DXA. Without both, tables render
incorrectly.

US Letter with 1 inch margins = 9360 DXA content width.

```typescript
// WRONG — missing cell widths:
new Table({
  columnWidths: [3120, 3120, 3120],
  rows: [...]
})

// CORRECT — both table AND cell widths in DXA:
new Table({
  width: { size: 9360, type: WidthType.DXA },
  columnWidths: [3120, 3120, 3120],
  rows: [new TableRow({
    children: [
      new TableCell({
        width: { size: 3120, type: WidthType.DXA },
        // ... content
      }),
      new TableCell({
        width: { size: 3120, type: WidthType.DXA },
        // ... content
      }),
      new TableCell({
        width: { size: 3120, type: WidthType.DXA },
        // ... content
      }),
    ]
  })]
})
```

Find every table in the DOCX renderer and fix the
column widths. The header table specifically: the
WBR title + week + prepared by table should be a
3-column table at 9360 total DXA:
- Column 1 (WBR badge + title): 5040 DXA
- Column 2 (week label): 2520 DXA
