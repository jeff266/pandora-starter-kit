## Part 2 — Deals table stage fields

Does the deals table have any stage
timestamp columns?

SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name = 'deals'
  AND (
    column_name ILIKE '%stage%'
    OR column_name ILIKE '%previous%'
    OR column_name ILIKE '%changed%'
    OR column_name ILIKE '%moved%'
    OR column_name ILIKE '%transition%'
  )
ORDER BY column_name;

## Part 3 — source_data JSONB inspection

If a source_data JSONB column exists on deals,
check what stage-related keys it contains for
Frontera deals:

SELECT DISTINCT jsonb_object_keys(source_data)
FROM deals
WHERE workspace_id = (
  SELECT id FROM workspaces
  WHERE name ILIKE '%frontera%' LIMIT 1
)
AND source_data IS NOT NULL
LIMIT 1;

Then specifically check if HubSpot stage history
is stored in source_data:

SELECT
  id,
  name,
  stage_normalized,
  source_data->'properties'->'dealstage' AS stage_field,
  source_data->'properties'->'hs_date_entered_closedwon'
    AS date_entered_closedwon,
  source_data->'properties'->'hs_date_entered_contractsent'
    AS date_entered_contract,
  source_data->'properties'->'hs_time_in_closedwon'
    AS time_in_closedwon
FROM deals
WHERE workspace_id = (
  SELECT id FROM workspaces
  WHERE name ILIKE '%frontera%' LIMIT 1
)
AND source_data IS NOT NULL
LIMIT 3;

HubSpot stores "hs_date_entered_[stage_id]" and
"hs_time_in_[stage_id]" properties for each stage.
If these are present in source_data, we have
implicit stage history even without a history table.

## Part 6 — Data volume check

How many Frontera deals have non-null source_data?

SELECT
  COUNT(*) AS total_deals,
  COUNT(source_data) AS with_source_data,
  COUNT(stage_normalized) AS with_stage,
  MIN(created_date) AS earliest_deal,
  MAX(updated_at) AS most_recent_update
FROM deals
WHERE workspace_id = (
  SELECT id FROM workspaces
