Bug 2: Legend shows "undefined", y-axis not formatted

In Image 1, the accepted chart shows:
- Legend label reads "undefined"
- Y-axis shows raw numbers (1800000, 1700000)
  instead of $1.8M / $1.7M

Both are QuickChart rendering issues in chart-renderer.ts.

Fix the legend label — the dataset needs a label field:
```typescript
// In buildChartJsConfig, bar case datasets:
datasets: [{
  label: chart.title,  // ADD THIS — fixes "undefined"
  data: values,
  backgroundColor: colors,
  borderRadius: 4,
}]
```

Apply the same `label: chart.title` to horizontal_bar
and line datasets.

Fix the y-axis formatting — the callback uses a
TypeScript typed parameter but QuickChart serializes
this to JSON so the function doesn't survive. Use
a string-based tick format instead:

QuickChart supports a special `tickFormat` string
or you can use the Chart.js `ticks.callback` as a
string representation. The cleanest fix for QuickChart
is to divide values by 1000 before sending and
label the axis with "($K)":

```typescript
// In buildChartJsConfig, before building config:
// If all values >= 1000, normalize to K
const maxValue = Math.max(...values);
const useKFormat = maxValue >= 1000;
const normalizedValues = useKFormat
  ? values.map(v => Math.round(v / 1000))
  : values;
const yAxisTitle = useKFormat
  ? (chart.y_label ? chart.y_label + ' ($K)' : '($K)')
  : (chart.y_label || '');

// Then use normalizedValues instead of values in datasets
// And set the y-axis title:
y: {
  grid: { color: 'rgba(0,0,0,0.06)' },
  border: { display: false },
  title: {
    display: yAxisTitle.length > 0,
    text: yAxisTitle,
    color: '#94A3B8',
    font: { size: 10 },
  },
  ticks: {
    color: '#64748B',
    font: { size: 11 },
    // Simple prefix — no callback function needed
    // Values are already in K
  },
}
```

Apply the same normalization to horizontal_bar x-axis.

For the donut chart, no axis so no change needed.
